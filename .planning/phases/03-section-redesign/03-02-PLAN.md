# Phase 3.2: Demo & Content Sections Redesign — Plan

## Objective

Redesign VoiceDemo, HowItWorks, and UseCases sections with purposeful brutalist styling. These sections explain what StrataBlue does—they should be clear, structured, and let content speak.

## Prerequisites

- Phase 2 complete (design tokens established)
- Plan 03-01 complete (Hero redesigned)
- Design utilities available

## Design Direction

### VoiceDemo: Clean Demo Cards
- Reduce visual noise, focus on audio interaction concept
- Remove duplicate card code (DRY principle)
- Purposeful animation (waveform shows state, not decoration)
- Consider: single demo card or tabbed interface

### HowItWorks: Primitive Step Layout
- Grid-based, not connector lines
- Large step numbers as primary visual
- Minimal icons, text-forward descriptions
- Clear visual hierarchy: number → title → description

### UseCases: Typographic Cards
- Icon de-emphasis, text emphasis
- Clear categorization
- Feature lists as data, not bullet decoration
